 ### 1. js里面的this的指向规则是什么？

this 的指向由函数的调用方式决定，常见的有以下几种情况：

1.	全局作用域：指向全局对象（浏览器中的 window）。
2.	普通函数调用：指向全局对象（非严格模式）或 undefined（严格模式）。
3.	对象方法调用：指向该对象。
4.	构造函数：指向新创建的对象实例。
5.	箭头函数：继承自外部作用域的 this。
6.	call、apply、bind：显式设置 this。
7.	事件处理函数：指向触发事件的 DOM 元素。

### 3.什么是闭包？

- 闭包（Closure）就是函数 + 其定义时的词法环境的组合；当一个函数在离开其定义处后被调用时，仍能访问外层作用域中的变量。（函数能够记住并访问它定义时的作用域，即使在它的外部环境已经执行完毕后）
- 作用/价值：做数据私有化与封装状态（不暴露到全局）、函数工厂（按配置生成函数）、缓存与控制副作用（如 once/memoize/防抖节流）、以及事件/异步回调中继续使用当时的上下文。
- 常见场景：计数器/权限控制、按钮点击携带参数、搜索联想的去抖/请求取消等。
- 注意点：for + var 导致共享同一绑定、异步里的“陈旧闭包”、以及长期持有大对象可能的内存泄漏（及时清理监听/计时器）。

```js
function createCounter(init = 0) {
  let count = init; // 私有变量：外部拿不到，只能通过返回的方法访问
   return {
    inc() { return ++count; },   // 这些方法形成闭包，持续访问到上面的 count
    dec() { return --count; },
    value() { return count; }
  };
}
const c = createCounter(1); // 使用
```

### 4.什么是事件循环？

- 事件循环是浏览器里调度 JS 与渲染的机制：在“取一个宏任务 → 执行 → 清空所有微任务 →（到点就）触发一帧渲染”的节奏中不停循环。
- JS 是单线程的，但网页还有计时器、网络、用户输入、渲染等一堆事。事件循环把这些事情排队、分时执行，既不阻塞页面，也保证顺序可预期。

- 事件循环又叫做消息循环，是浏览器渲染主线程的工作方式。在 Chrome 的源码中，它开启一个不会结束的 for 循环，每次循环从消息队列中取出第一个任务执行，而其他线程只需要在合适的时候将任务加入到队列未尾即可。
- 过去把消息队列简单分为宏队列和微队列，这种说法目前已无法满足复杂的浏览器环境，取而代之的是一种更加灵活多变的处理方式。
- 根据 W3C 官方的解释，每个任务有不同的类型，同类型的任务必须在同一个队列，不同的任务可以属于不同的队列。不同任务队列有不同的优先级，在一次事件循环由浏览器自行决定取哪一个队列的任务。但浏览器必须有一个微队列，微队列的任务一定具有最高的优先必须优先调度执行。

### 5.如何理解JS中的异步？

- JS是一门单线程的语言，这是因为它运行在浏览器的渲染主线程中。渲染主线程只有一个，而渲染主线程承担着诸多的工作，渲染页面、执行 JS 都在其中运行。
- 如果使用同步的方式，就极有可能导致主线程产生阻塞，从而导致消息队列中的很多其他任务无法得到执行。这样一来，一方面会导致繁忙的主线程白白的消耗时间，另一方面导致页面无法及时更新，给用户造成卡死现象。
- 所以浏览器采用异步的方式来避免。具体做法是：当某些任务发生时，比如计时器、网络、事件监听，主线程将任务交给其他线程去处理，自身立即结束任务的执行，转而执行后续代码。当其他线程完成时，将事先传递的回调函数包装成任务，加入到消息队列的末尾排队，等待主线程调度执行。在这种异步模式下，浏览器永不阻塞，从而最大限度的保证了单线程的流畅运行。


### 6. JS 基本类型 （或者数据类型和区别）

  类型分类：
 - JS有八大数据类型，八大数据类型又可以分为基本数据类型和应用数据类型。
 - 基本数据类型有string、number、undefined、null、boolean、bigint、symbol。
 - 应用数据类型为object，比如普通对象、数组、函数、Date、RegExp、Map、Set等等。
 - 
  区别：
 - （存储方式不同）基本类型存的就是值本身，存放在栈(stack)中；引用类型存在栈中的是地址（引用），真实数据存在堆（heap）中。
 - （赋值和传参行为不同）基本类型是值拷贝，互不影响；引用类型拷贝的是引用，多个变量可能指向同一个对象。
 - （比较方式）基本类型比较的是值是否相等，引用类型比较的是引用（地址）是否相同。

  注意：
   1. 所有数字（整数、小数、NaN、Infinity）都是 number。
   2.	NaN：表示“不是一个有效数字”，但类型依然是 number 。```typeof NaN === 'number'```
   3. 变量声明了但没赋值，就是 undefined；访问对象上不存在的属性。
   4. null 表示一个刻意置空的空对象值（这里本来应该有个对象，但现在还没有）。
   5. ```typeof null === 'object'```（历史遗留 bug）.
   6. symbol ,ES6新增，用来创建独一无二的值，常用来做对象的私有属性key。
   7. 字符串是不可变的（immutable）。
       ```js
       ///
       let str = 'abc'
       str[0] = 'Z'
       console.log(str)   // 仍然是 'abc'
       ///
       let s = 'abcde'
       let s2 = s.toUpperCase()
       console.log(s)   // 'abcde'
       console.log(s2)  // 'ABCDE'
       ```
